# 關聯客戶

找出有所關聯的被保人。方法有以下三種:
1. 直接連接: 彼此互為要保人v.s.被保人的關係。
2. 透過要保人連接: 彼此之間沒有直接連接，但擁有相同的要保人。
3. 透過標的物連接: 彼此之間沒有直接連接，但擁有相同的標的物。

## 筆記:
1. 有要保人從來沒有以被保人身分買過保單的; e.g. 5931a0575347fd7c392fc2dac8df3ac1e33ad03b

## 目前考慮:

1. 被保人
2. 要保人
3. 標的物

## 使用資料:

* 時間:
	2017/01/01 ~ 2022/03/31

* 險別:
	1. 車險(包含強制、任意)
	2. Hth(**不包含團傷**) : 因關注的是"人與人的關係"，因此包括防疫險、排除團傷
	**📝note: 團旅要小心**

## 關係:

1. 直接相連: 關係 = 要保人 v.s. 被保人
2. 透過要保人: 關係 = 被保人 v.s. 被保人 (兩者之間至少有一個相同要保人)
3. 透過標的物: 關係 = 被保人 v.s. 被保人 (兩者之間至少有一個相同標的物)

** 以上關係皆不分年 **

# 延伸: 關聯險種

## 目標:

透過不同險種之間的關係強度，並找出容易連接到其他險種的重要節點。

>e.g.  跨險別的關係中有較強連結的範例:
>- 汽車第三人責任保險-體傷(含多倍) --|3,681|-- 金如意團體傷害保險 
>  (但其實也只有3,681 / 765,910 = 0.6%)
>-  第三人責任保險機車乘客責任附加條款 --|125|-- 保滿意個人傷害保險 (125 / 5535 = 2.3%)

* 時間:
	先以2020年為例。

* 險別:
	1. 車險(包含強制、任意)
	2. Hth(**包含團傷**) : 因關注的是險種 v.s. 險種，所以考慮團傷 

* 欄位說明:
	* 重要度:
	> degree = 連結了多少其他節點? = 連接的節點數(險種數) / (所有險種數-1)

	* 傳播能力: 
	> betweeness = 位在多少"最短路徑上"? = 經過此點的最短路徑 / 所有最短路徑
	> closeness = 距離其他點多近? = 1 / sum(此點到所有鄰居的最短路徑長度)

	* 重要鄰居 (只看其他險別) :
	> top{x} = 第x多相同被保人的鄰居

📝note: 跨險別的時候，險種看到多細? e.g. 車險 v.s 各傷到幾碼?

* 欄位說明:
	**links** = 營業 --> |被保人| --> 醫院 = **不重複  營業 --> (被保人+院所)**
	
		`|被保人| --> 醫院` : 同一位客戶不會被重複計算，
			e.g. 小明去診所A就醫兩次，只會算一次 (不論是否同一個賠案)
			
	**clm_idHospotal = 該營業人員的所有出險客戶的總就醫次數**
	
		營業--(被保人+院所):
			e.g. 營業人員甲只有兩位客戶，小明 & 小黑，
				(note: 不管照訪幾次，都只會產生一個不重複的 *營業--(被保人+院所)*)
				小明曾照訪5次診所A & 10次醫院B，共產生兩個links，:
					營業人員甲--小明--診所A
					營業人員甲--小明--醫院B 
				小黑曾照訪5次診所A，1個links，:
					營業人員甲--小黑--診所A
				==> 營業人員甲共計有**3個不重複的 *營業--(被保人+院所)***
				
	**hospital_ratio  (_of hospital_i_) = links (_in hospital_i_)  / clm_idHospotal**
	
		在同一間院所的就診比例
			e.g. 接續營業人員甲的範例:
				診所A: 共計有2個links --> 67%在診所A就診
				診所B: 共計有1個links --> 33%在醫院B就診
